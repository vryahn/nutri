Decisiones, no solo features

Cada trade-off se resuelve a favor de la exactitud de los datos. Estas son las decisiones que dieron forma al sistema.

Integridad de datos

La exactitud gana todo trade-off

Todo se almacena por 100 g y se calcula en vistas SQL al leer — los valores nunca se copian entre tablas. El dato dudoso se marca ⚠, nunca se guarda en silencio.

El kcal declarado se contrasta con los factores de Atwater (4/4/9, fibra a 2 kcal/g): una diferencia mayor al 25% exige revisión antes de guardar.

IA con paracaídas

Cascada de respaldo, custodiada por evals

La transcripción de etiquetas desde foto corre sobre keys de free tier. Cualquier error — timeout, cuota, JSON inválido — cae al siguiente modelo sin que el usuario lo note.

Un golden set puntuado de fotos reales de etiquetas corre antes de cada cambio a la IA: cualquier regresión contra el baseline bloquea el push.

gemini-3.5-flash ✗ timeout → gemini-2.5-flash ✗ 429 → mistral-small ✓
✓ npm run eval — regresión vs baseline = no hay deploy

El schema de Gemini se traduce a JSON Schema estricto para Mistral — misma salida estructurada, otro dialecto.

Búsqueda

Búsqueda semántica del catálogo

Recuperación híbrida: la coincidencia exacta responde primero; si se queda corta, pgvector completa con similitud coseno sobre embeddings de 768 dims generados al guardar cada alimento.

La misma infraestructura alimenta «Pregúntale a tu bitácora»: un pipeline RAG de 3 pasos que responde citando solo datos realmente registrados.

Seguridad

Privacidad por diseño

Row Level Security aísla a cada usuario dentro de Postgres: el filtro vive en la base de datos, no en la UI. Ninguna secret key viaja al cliente.

El repo es público con cero credenciales en el historial — el diseño asume que cualquiera puede leer el código.

create policy foods_select
  on nutri.foods for select
  using (owner = auth.uid());
sesión: usuario A Yogur griego Avena Café de olla
filas del usuario B ████████████ ██████████ █████████████

Postgres ni siquiera devuelve las filas del otro usuario — no es un filtro de UI que un bug pueda saltarse.

Interoperabilidad

Su propio conector MCP

Un servidor MCP remoto con OAuth expone 8 tools — registrar, buscar, auditar — actuando con la identidad de cada usuario, bajo el mismo RLS.

La bitácora se volvió conversacional: registrar una comida o auditar una semana ocurre desde cualquier cliente MCP — Claude, ChatGPT, Perplexity y más — sin pasar por la UI.

Sistema de diseño

Una paleta que nunca miente

El acento nunca ocupa el hue de un dato — los carbohidratos viven en índigo para que «botón primario» y «proteína» no signifiquen el mismo color.

Cada tema recalcula la luminancia en OKLCH contra su fondo más adverso: todo par texto/fondo se mantiene en 4.5:1 o más.
